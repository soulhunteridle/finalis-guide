What Are These Stats?

There are 3 separate ignore defense related stats. They all do different things:

  • Ignore Defense — a flat number. This is subtracted from enemy defense before damage is calculated.
  • Ignore Defense DMG — a multiplier that scales how much your ignore defense actually does. More on this below.
  • Ignore Defense % — reduces enemy defense by a percentage first, before your Ignore Defense number is applied.
Why does this matter? Enemy defense directly reduces your damage. The more you can bypass it, the more damage you deal. Understanding how these stats interact tells you exactly how much a deco upgrade is actually worth.
📉
Part 1 — When Ignore Defense < Enemy Defense
The Formula

When your Ignore Defense hasn't fully overcome enemy defense yet, the damage ratio between two setups is:

Dmg Ratio = (New Ignore Def / Old Ignore Def) × (New Ignore Def DMG / Old Ignore Def DMG)
New Damage = Old Damage × Dmg Ratio

If Ignore Def DMG doesn't change between the two setups, that factor = 1 and you can drop it.

Example Calculations

Test environment: Astarte hit damage, enemy defense = 1,823,127, Ignore Def DMG = 625 (unchanged).

Pair 1 — Row 1 → Row 2
Ignore Def: 1,648,696 → 1,687,326 | Dmg: 311 → 319

Formula ratio = (1,687,326 / 1,648,696) × (625 / 625)
               = 1.02343 × 1
               = 1.02343

Actual ratio   = 319 / 311 = 1.02572 ✓
Pair 2 — Row 2 → Row 3
Ignore Def: 1,687,326 → 1,743,233 | Dmg: 319 → 329

Formula ratio = (1,743,233 / 1,687,326) × (625 / 625)
               = 1.03313 × 1
               = 1.03313

Actual ratio   = 329 / 319 = 1.03135 ✓

Small differences at the 3rd decimal point are just display rounding in-game. At the actual damage scale these are negligible.

Part 2 — Ignore Defense % (and Why It's Weak)
How It Works

Ignore Defense % applies as the first reduction to enemy defense — before your Ignore Defense number even comes into the picture.

Effective Enemy Def = Enemy Def × (1 − Ignore Def %)
Example: Enemy Def 100, your Ignore Def % = 10% → Effective Enemy Def = 90
  • Using it on a 0 defense enemy (Training Cave) does nothing — 0 × anything = 0
  • Can exceed 100% via passives like Defense Ignore and Rune Beginner Mind — tested at 120%
  • Enemy defense likely cannot go negative even at 120%+ — it appears to floor at 0
Why It's Not Worth Investing In

Back to math 101 — this stat reduces an already-small number. Our Ignore Defense values are already enormous, especially factored with the Otherworld multiplier feature. Enemy defense is already being obliterated by raw Ignore Def alone.

⚠ Ignore Defense % is not a recommended investment. The returns are minimal given how much Ignore Defense we already have.

Part 3 — When Ignore Defense > Enemy Defense
The Formula Tweak

Once your Ignore Defense surpasses enemy defense, the raw Ignore Def values no longer matter — what matters is the gap above enemy defense. The formula becomes:

Dmg Ratio = ((New Ignore Def − Enemy Def) / (Old Ignore Def − Enemy Def)) × (New Ignore Def DMG / Old Ignore Def DMG)
When enemy def = 0: formula is identical to Part 1 (subtracting 0 changes nothing)
Example Calculations

Test environment: Astarte hit damage on Training Cave (enemy def = 0). Since enemy def = 0, the formula simplifies to the same as Part 1.

Pair 1 — Row 1 → Row 2 (enemy def = 0)
Ignore Def: 1,648,696 → 1,743,233 | Ignore Def DMG: 625 → 625 | Dmg: 35 → 37

Formula ratio = (1,743,233 / 1,648,696) × (625 / 625)
               = 1.05733 × 1
               = 1.05733

Actual ratio   = 37 / 35 = 1.05714 ✓
Pair 2 — Row 8 → Row 9 (enemy def = 0)
Ignore Def: 109,400,000 → 119,280,000 | Ignore Def DMG: 200,670 → 273,625 | Dmg: 780,000 → 1,120,000 (AF)

Formula ratio = (119,280,000 / 109,400,000) × (273,625 / 200,670)
               = 1.09030 × 1.36359
               = 1.48672

Actual ratio   = 1,120,000 / 780,000 = 1.43590 ≈ ✓

Pair 2 is slightly wider (~3.5% off) due to display rounding when numbers jump between unit scales. The principle still holds.

Practical Use — Predicting Your New Damage

Before committing to a deco upgrade, you can use the formula below to predict how much additional damage you will get. Check the change by using: the number in your character level up ignore defense as base + deco stats x otherworld multiplier:

  1. The number in your character level up ignore defense as base
  2. The change between your current deco and your targetted deco
  3. The multipler on your otherworld stats
  4. Use data from 1 to 3 to estimate your current ignore def and ignore def dmg, and your next ignore def and ignore def dmg
  5. Run the formula and see the ratio

New Damage = Old Damage × (New Ignore Def / Old Ignore Def) × (New Ignore Def DMG / Old Ignore Def DMG)

Works for both upgrades and downgrades. Just swap old/new values accordingly.

Part 4 — The Threshold Problem

Right at the moment your Ignore Defense crosses enemy defense, the formula becomes unreliable. Here's exactly what happens — tested with Smash damage, enemy def = 872.

Stage 1 — Below Threshold (Consistent) ✓
Ignore Def < 872 — Formula works perfectly
Row 1→2: (871.51 / 870) × 1 = Formula: 1.00174 | Actual: 1.00174 ✓ Perfect
Row 2→3: (872 / 871.51) × 1 = Formula: 1.00056 | Actual: 1.00056 ✓ Perfect
Stage 2 — Just Crossed Threshold (Inconsistent) ✗

Once Ignore Def just barely exceeds 872, the formula switches to using the gap — but the gap is so tiny that small rounding differences create enormous ratio errors.

Ignore Def just above 872 — Formula breaks down
Row 4→5: ((873.99−872)/(873−872)) × 1 = Formula: 1.990 | Actual: 1.010 ✗ Way off
Row 5→6: ((874.99−872)/(873.99−872)) × 1 = Formula: 1.503 | Actual: 1.010 ✗ Still off

The gap (e.g. 1.99, 2.99) is so small that another variable seems to take over the calculation. This is an unresolved behaviour — see Part 5.

Stage 3 — Gap Widens, Formula Recovers ≈

As Ignore Def grows far beyond enemy def, the gap becomes large enough that the formula converges back toward accuracy.

Ignore Def far above 872 — Converging back
Row 10→11: ((10577−872)/(7333−872)) × (15/7) = Formula: 3.219 | Actual: 3.000 ≈ Getting closer
Row 11→12: ((15443−872)/(10577−872)) × (30/15) = Formula: 3.003 | Actual: 2.905 ≈ Very close (~3.3% off)
Why this doesn't matter in practice: Our Ignore Defense already surpasses enemy defense by an enormous margin — especially with deco upgrades and Otherworld multipliers. We are never sitting at the threshold. The formula is accurate where it counts.

Part 5 — Unanswered Questions

These are open questions from testing. They haven't been pursued further because the formula is already proven accurate in practical scenarios and the errors only occur in negligible edge cases.

What happens when Ignore Def DMG increases while Ignore Def < Enemy Def?
Very unlikely to encounter in practice — our Ignore Defense is already enormous even with low deco.
What really happens at the exact moment enemy defense is crossed?
Another variable or calculation seems to kick in temporarily, but becomes irrelevant as the gap widens.
Can enemy defense reach negative numbers with Ignore Def % > 100%?
Likely floors at 0 but unconfirmed. Doesn't matter either way given our Ignore Def gap.
How does the formula handle Ignore Def DMG changing from 0 to 2 as the initial value?
Assumed to be ×2 using 0 as base 1, since the first deco gives 2 Ignore Def DMG not 1. Only relevant at Special rarity — near zero practical impact.

TLDR
Summary
  1. When Ignore Def < Enemy Def: Dmg Ratio = (New / Old Ignore Def) × (New / Old Ignore Def DMG)
  2. When Ignore Def > Enemy Def: same formula but use the gap above enemy def, not raw values
  3. When enemy def = 0 (Training Cave): both formulas are identical — subtract 0, nothing changes
  4. Formula breaks down right at the threshold — unpredictable zone, but irrelevant in practice
  5. Ignore Defense %: not worth investing — reduces an already obliterated number
  6. Use the formula to predict damage before committing to any deco upgrade
